# ヘンな将棋

7×7の不規則な盤面で遊ぶ、1台2人用のターン制ボードゲーム試作です。プレイヤーは相手に見せずに8ポイント以内の駒を編成し、自陣へ置く駒と初期持ち駒にする駒を選びます。

オンライン通信、AI、アカウント、課金、ランキングは使用しません。

## 起動方法

Flutter安定版をインストールした環境で実行します。

```sh
flutter pub get
flutter run
```

対象端末を指定する例:

```sh
flutter run -d chrome
flutter run -d android
flutter run -d ios
```

iOSビルドにはmacOSとXcodeが必要です。Webのリリースビルドは `flutter build web` で生成できます。

## 遊び方

1. 30〜40マスのランダム盤面を確認する（対局開始前は再生成可能）
2. ランダムに決まった先手・後手を確認する
3. 後手が上側の陣営Aまたは下側の陣営Bを選ぶ
4. 先手が8ポイント以内で秘密編成し、自陣への配置と初期持ち駒を決める
5. 受け渡し画面で端末を後手へ渡す
6. 後手も秘密編成と配置を行う
7. 駒を公開し、先手から交互に移動または駒打ちを行う
8. 相手の王将を取るか、相手が投了すると勝利

王将は無料で必ず1枚含まれ、自陣の盤上へ配置する必要があります。王将以外はすべて初期持ち駒にしても構いません。取得した敵駒は自分向きの持ち駒になります。

盤上では自分の駒、または手番プレイヤーの持ち駒を選ぶと合法マスが緑色になります。黄色は選択中、橙色は直前の移動元・移動先です。陣営Aの駒は青色と下向き印、陣営Bの駒は赤色と上向き印で区別しています。

## 駒とコスト

| 駒 | 表示 | コスト |
|---|---:|---:|
| 王将 | 王 | 0（必須） |
| 飛車 | 飛 | 5 |
| 角 | 角 | 4 |
| 金 | 金 | 3 |
| 銀 | 銀 | 2 |
| 桂馬 | 桂 | 2 |
| 香車 | 香 | 2 |
| 歩 | 歩 | 1 |

王将以外の合計は8ポイントまでです。同種の駒を複数選択できます。動きは通常の将棋と同じで、前方は先手・後手ではなく選択した陣営によって決まります。

## 不規則盤面と駒打ち

盤面生成はUIから独立した純粋Dartロジックです。`BoardGenerator.generate(seed)` に整数シードを渡すと、同じ使用可能マスを再現できます。現在のシードは盤面確認画面に表示されます。

生成盤面は次を検証します。

- 使用可能マスが30〜40マス
- 全使用可能マスが上下左右で接続
- 上下それぞれの初期配置エリアが最低5マス
- 多数の幅1マス通路を避けるため、65%以上のマスが上下左右に2つ以上の隣接マスを持つ
- 左右対称ではない

駒打ちは空いている使用可能マスに限ります。二歩は禁止です。また、不規則盤面に合わせ、空の盤面でも次の移動先が1つもない場所への駒打ちを全駒共通で禁止しています。打ち歩詰めは判定しません。

## 試作で省略したルール

- 成り
- 王手、詰み、王手放置の判定
- 打ち歩詰め
- 千日手、持将棋、入玉
- 時間制限

王手判定を行わないため、王将を危険なマスへ移動できます。勝敗は王将を実際に取得した時点で決まります。取得された王将は持ち駒には加わりません。

## 実装構成

- `lib/models/`: null safeなゲームデータモデル
- `lib/data/`: コストとデータ駆動の移動パターン
- `lib/game/`: 盤面生成・検証、編成検証、移動、駒打ち、勝敗の純粋ロジック
- `lib/providers/`: Riverpodによるゲーム進行と状態更新
- `lib/screens/`, `lib/widgets/`: 12フェーズのUIとWidget盤面
- `test/`: ロジック、Riverpod進行、UIのテスト

移動は `MovementPattern` の相対座標とスライド可否で定義しています。ナイト、ビショップ、ルーク、クイーンなどは駒定義と表示を追加する形で拡張できます。将来的には成り状態、王手判定、棋譜、シード直接入力UIをゲームロジックを保ったまま追加できます。
